Case-sensitivity in accesskey test

This tests how accesskey handles case, and repeated instances of the same value, in order to deal with issues raised in issue 485.

The test is run with keyboard that has "a" and "b" keys. If you cannot activate or focus any links, then (probably) the accesskey attribute is ineffective and the results are not helpful.

1. "A"

The following link uses accesskey="&#x0041;". Can you use accesskey A to focus or activate it?

Results:

  • Blink, Webkit, Gecko / MacOS 10.10 / australian keyboard: No
  • Gecko / MacOS 10.10 / australian keyboard: ctrl-alt-a focuses link

2. "a"

The following link to "page one" uses accesskey="&#x0061;". Can you use accesskey a to focus or activate it?

Results:

  • Blink, Webkit, Gecko / MacOS 10.10 / australian keyboard: No
  • Gecko / MacOS 10.10 / australian keyboard: ctrl-alt-a twice focuses link

3. "a"

The following link uses accesskey="&#x0061;". Can you use accesskey a to focus or activate it?

Results:

  • Blink, Webkit / MacOS 10.10 / australian keyboard: No
  • Gecko / MacOS 10.10 / australian keyboard: ctrl-alt-a thrice focuses link

4. "A"

The following link uses accesskey="&#x0041;". Can you use accesskey A to focus or activate it?

Results:

  • Blink, Webkit / MacOS 10.10 / australian keyboard: ctrl-alt-a or ctrl-alt-shift-a activate the link
  • Gecko / MacOS 10.10 / australian keyboard: ctrl-alt-a 4 times focuses the link

5. "b"

The following link uses accesskey="&#x0062;". Can you use accesskey b to focus or activate it?

Results:

  • Blink, Webkit, Gecko / MacOS 10.10 / australian keyboard: Yes
